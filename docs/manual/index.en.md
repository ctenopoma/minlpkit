# User Manual

A manual for running through the **observe (analyze) → diagnose (findings/recipe) → improve (transforms/frameworks) → verify (compare_variants)** pipeline in minlpkit all at once. All code examples work via copy-paste (verified by doctest / demo in this repository).

## Setup

### Using within the repository

```powershell
uv sync
$env:PYTHONIOENCODING = 'utf-8'   # If Japanese text gets garbled in the console
uv run python demo.py
```

### Using from an external project

`minlpkit` is a distribution package that works independently (core runtime dependencies are only pyscipopt/pandas/numpy/scipy). Live visualization (`minlpkit.live`) and tuning (`minlpkit.tune`) are **extras** that require additional dependencies (`viz` = flask/plotly/kaleido, `tune` = optuna). If you only need the core, extras are unnecessary.

```powershell
# Install via GitHub (Latest version)
uv add git+https://github.com/ctenopoma/minlpkit.git                          # Core only (4 dependencies)
uv add "minlpkit[viz,tune] @ git+https://github.com/ctenopoma/minlpkit.git"   # + Live visualization / Tuning
```

```python
import minlpkit as mk   # With just this, analyze / compare_variants / transforms / frameworks are available
# If you install extras:
from minlpkit.live import solve_with_monitor, RunLogger   # Requires minlpkit[viz]
from minlpkit.tune import tune                            # Requires minlpkit[tune]
```

If you import `minlpkit.live` / `minlpkit.tune` without installing the extras, an `ImportError` (such as `uv add "minlpkit[viz]"`) will be raised to guide you on how to install them.
